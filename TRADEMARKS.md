# dukememory. Brand and Trademark Guidelines

The `dukememory` name, wordmark, screenshots, UI identity, and project branding
identify the original project by danilkryachko.

The source code is licensed under Apache-2.0. That license lets you use,
modify, and distribute the code, including commercially. It does not grant
permission to use the `dukememory` brand in a way that suggests affiliation,
endorsement, sponsorship, or that a fork is the original project.

## Allowed

- Refer to the project as `dukememory` when linking to or discussing the
  original repository.
- State that your project is based on, compatible with, or forked from
  `dukememory`, as long as the statement is accurate.
- Use the name in dependency notices, attribution, changelogs, or technical
  documentation.

## Not Allowed Without Written Permission

- Use `dukememory` as the name of a fork, hosted service, product, package,
  company, plugin, marketplace item, or commercial offering.
- Use project screenshots, UI styling, logos, badges, or visual identity to
  market a derivative product or service.
- Present a fork or derivative as the official `dukememory` project.
- Imply endorsement, partnership, sponsorship, or maintenance by danilkryachko.

## Forks and Derivatives

Forks should use a clearly different name and should disclose their relationship
to the original project. A clear example:

```text
This project is a fork of dukememory and is not affiliated with or endorsed by
danilkryachko.
```

For permission to use the brand in a product, service, integration, or public
launch, contact danilkryachko.
